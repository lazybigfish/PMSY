# 功能模块统计卡片需求

## 需求概述
在功能模块页面的层级树上方增加统计卡片，动态展示各层级模块的数量和完成度统计。

## 现状问题
当前功能模块页面直接展示模块树，用户无法快速了解：
1. 项目有多少个各级模块
2. 各层级模块的整体完成度
3. 项目模块的层级深度

## 需求详情

### 1. 统计卡片位置
在「功能模块」标题和按钮栏下方，模块树上方增加一行统计卡片。

### 2. 动态层级展示
根据实际模块层级深度动态展示卡片：
- 如果只有一级模块 → 只展示「一级模块」统计
- 如果有三级模块 → 展示「一级模块」「二级模块」「三级模块」统计
- 如果有五级模块 → 展示到「五级模块」统计

### 3. 每张卡片展示内容
每张统计卡片包含：
- **标题**：X级模块（如：一级模块、二级模块）
- **数量**：该层级模块总数
- **完成度**：显示百分比数字

### 4. 完成度计算规则
- 模块状态为 `completed` 视为已完成
- 完成度 = 已完成模块数 / 该层级总模块数 × 100%

### 5. 视觉设计（卡片即进度条）
- **卡片即进度条**：整个卡片底部有一个背景色填充层，根据完成度百分比填充
- 填充色使用渐变色（从 primary-500 到 violet-500）
- 卡片横向排列，支持横向滚动（当层级较多时）
- 每张卡片有清晰的层级标识和数量显示
- 卡片有 hover 效果
- 文字内容在填充层上方清晰可读

## 技术方案
1. 修改 `src/pages/projects/tabs/FunctionalModules.tsx`
2. 添加统计计算函数，遍历模块树统计各层级数据
3. 在模块树上方添加统计卡片组件
4. 使用动态渲染，根据实际层级数量生成对应卡片

- [ ] 根据实际模块层级动态展示对应层级的统计卡片
- [ ] 每张卡片正确显示该层级的模块数量
- [ ] 每张卡片正确显示该层级的完成度百分比和进度条
- [ ] 完成度计算准确（基于 completed 状态）
- [ ] 卡片样式美观，有 hover 效果
