# 日期选择器优化需求

## 需求概述
优化全局日期选择控件，提升用户体验，增加快捷日期选择功能。

## 现状问题
1. 当前使用原生 `type="date"` 输入框，必须点击日历图标才能打开日期选择器
2. 选择日期过程较为繁琐，没有快捷选项

## 需求详情

### 1. 交互优化
- **点击区域扩大**：点击整个日期输入框区域即可打开日期选择弹窗，而不是只能点击日历图标
- **弹窗形式**：使用自定义弹窗替代浏览器原生日期选择器

### 2. 快捷日期功能
在日期选择弹窗顶部增加快捷选项按钮：
- 今天
- 明天
- 三天后
- 一周后
- 一月后
- 三月后

点击快捷按钮后自动计算对应日期并填充，同时关闭弹窗。

### 3. 日历功能
- 月份切换（左右箭头）
- 显示当前月份日历网格
- 高亮选中日期
- 标识今天日期
- 支持清除已选日期

### 4. 视觉设计
- 与现有系统UI风格保持一致
- 使用圆角、阴影、过渡动画
- 选中状态使用主题色（primary-500）

## 影响范围
需要替换以下页面中的日期选择器：
1. `src/pages/tasks/TaskCreatePage.tsx` - 创建任务页面
2. `src/pages/tasks/components/TaskCreateModal.tsx` - 创建任务弹窗
3. `src/pages/tasks/components/TaskFilterBar.tsx` - 任务筛选栏
4. `src/pages/tasks/components/AdvancedSearch.tsx` - 高级搜索
5. `src/pages/projects/components/SupplierDetailModal.tsx` - 供应商详情
6. `src/pages/projects/reports/ReportEditor.tsx` - 报告编辑
7. `src/pages/projects/tabs/Reports.tsx` - 项目报告

## 技术方案
1. 创建全局 `DatePicker` 组件：`src/components/DatePicker.tsx`
2. 组件接受 `value`、`onChange`、`placeholder`、`required` 等属性
3. 使用 React state 管理弹窗显示/隐藏
4. 使用 useEffect 监听点击外部自动关闭弹窗
5. 日期计算使用原生 Date API

## 验收标准
- [ ] 点击输入框任意位置都能打开日期选择弹窗
- [ ] 快捷日期按钮（今天、明天、三天后、一周后、一月后、三月后）正常工作
- [ ] 日历可以正常切换月份
- [ ] 可以正常选择日期并回填到输入框
- [ ] 可以清除已选日期
- [ ] 弹窗外点击可以关闭弹窗
- [ ] 所有使用日期选择的地方都已替换为新组件
