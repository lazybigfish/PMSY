# 项目列表交互优化需求

## 需求概述
优化项目管理列表的交互方式，减少操作栏的功能交叉，将不同功能分散到对应的列上，提升用户体验。

## 现状问题
当前操作栏包含多个功能按钮（模块、里程碑、详情、删除），导致：
1. 操作栏过于拥挤，功能交叉
2. 用户需要思考应该点击哪个按钮
3. 不符合直觉的操作流程

## 需求详情

### 1. 操作栏简化
**操作栏只保留两个按钮：**
- **详情按钮** - 进入项目详情页
- **删除按钮** - 删除项目（仅项目经理可见）

### 2. 列级点击交互
将原本操作栏的功能分散到对应列：

| 列 | 点击行为 | 跳转目标 |
|---|---|---|
| **项目信息列**（整列可点击） | 点击进入项目详情 | `/projects/${id}` |
| **当前阶段列**（标签可点击） | 点击进入里程碑页面 | `/projects/${id}?tab=milestones` |
| **进度列**（进度条可点击） | 点击进入功能模块页面 | `/projects/${id}?tab=modules` |
| **健康度列**（图标可点击） | 点击进入风险页面 | `/projects/${id}?tab=risks` |

### 3. 视觉反馈
- 可点击区域需要有明显的 hover 效果
- 鼠标悬停时显示手型光标（cursor-pointer）
- 适当的颜色变化提示可点击性

### 4. 交互细节
- **项目信息列**：点击项目名称、客户信息或整行区域都进入详情
- **当前阶段标签**：保持原有样式，增加点击事件
- **进度条**：保持原有样式，增加点击事件
- **健康度图标**：保持原有样式，增加点击事件

## 技术方案
1. 修改 `src/pages/projects/ProjectList.tsx`
2. 为各列添加点击事件处理器
3. 使用 `useNavigate` 进行页面跳转
4. 添加适当的 CSS hover 效果

## 验收标准
- [ ] 操作栏只显示"详情"和"删除"两个按钮
- [ ] 点击项目信息列任意位置进入项目详情
- [ ] 点击当前阶段标签进入里程碑页面
- [ ] 点击进度条进入功能模块页面
- [ ] 点击健康度图标进入风险页面
- [ ] 所有可点击区域都有明显的 hover 效果
- [ ] 删除按钮功能保持正常（仅项目经理可见）
