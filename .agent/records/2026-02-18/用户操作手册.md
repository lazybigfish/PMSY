## 一、项目基本介绍

### 1.1 系统概述

**PMSY（Project Management System）** 是一款专为工程项目管理设计的综合管理平台，旨在帮助企业实现项目全生命周期的数字化管理。

```
┌─────────────────────────────────────────────────────────────┐
│                      PMSY 项目管理系统                        │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│   ┌──────────┐  ┌──────────┐  ┌──────────┐  ┌──────────┐   │
│   │ 工作台   │  │ 项目管理 │  │ 任务中心 │  │ 相关方   │   │
│   └──────────┘  └──────────┘  └──────────┘  └──────────┘   │
│                                                             │
│   ┌──────────┐  ┌──────────┐  ┌──────────┐                 │
│   │ 水漫金山 │  │ 系统设置 │  │ 个人中心 │                 │
│   └──────────┘  └──────────┘  └──────────┘                 │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 1.2 系统核心能力

| 能力模块 | 功能说明 | 适用场景 |
|---------|---------|---------|
| **项目全生命周期管理** | 从立项到结项的完整流程管理 | 工程项目跟踪 |
| **任务协同与跟踪** | 任务分配、进度跟踪、看板视图 | 团队协作 |
| **里程碑管理** | 关键节点设置、进度监控 | 项目进度把控 |
| **风险预警** | 风险识别、等级评估、预警通知 | 风险管控 |
| **供应商管理** | 供应商信息、合作记录、评价 | 供应链管理 |
| **客户管理** | 客户信息维护、项目关联 | 客户关系管理 |

### 1.3 用户角色体系

```
                    ┌──────────────┐
                    │   系统管理员  │
                    │    Admin     │
                    └──────┬───────┘
                           │
           ┌───────────────┼───────────────┐
           │               │               │
    ┌──────▼──────┐ ┌──────▼──────┐ ┌──────▼──────┐
    │  项目经理   │ │  团队成员   │ │  普通用户   │
    │   Manager   │ │ Team Member │ │    User     │
    └─────────────┘ └─────────────┘ └─────────────┘
```

| 角色 | 权限范围 | 主要职责 |
|-----|---------|---------|
| **系统管理员** | 全部权限 | 系统配置、用户管理、数据维护 |
| **项目经理** | 项目级权限 | 项目创建、任务分配、进度监控 |
| **团队成员** | 任务级权限 | 任务执行、进度更新、协作沟通 |
| **普通用户** | 查看权限 | 查看项目、浏览任务 |

### 1.4 系统访问方式

- **系统地址**：请通过企业内部提供的链接访问
- **支持浏览器**：Chrome、Edge、Firefox、Safari（推荐 Chrome）
- **移动端支持**：支持手机、平板访问（部分功能可能受限）

## 二、用户账号管理规范

### 2.1 账号注册与开通

#### 2.1.1 账号开通流程

```
┌──────────┐    ┌──────────┐    ┌──────────┐    ┌──────────┐
│ 提交申请  │ → │ 主管审批  │ → │ 管理员开通 │ → │ 接收账号  │
└──────────┘    └──────────┘    └──────────┘    └──────────┘
```

**开通方式**：
1. **管理员直接开通**：联系系统管理员提供必要信息
2. **自助注册**：如系统开放注册功能，按页面提示完成

#### 2.1.2 首次登录须知

> ⚠️ **重要提示**：首次登录后请立即修改初始密码！

**首次登录步骤**：
1. 访问系统登录页面
2. 输入用户名（非邮箱）和密码
3. 点击「登录」进入系统
4. 进入「个人设置」修改密码
5. 完善个人资料信息

### 2.2 账号安全规范

#### 2.2.1 密码安全要求

| 要求项 | 规范说明 | 示例 |
|-------|---------|------|
| **长度** | 至少 8 位字符 | `Willyou@2026` |
| **复杂度** | 包含大小写字母、数字、特殊符号 | `Pmsy@2026#` |
| **更换周期** | 建议每 90 天更换一次 | - |
| **禁止行为** | 不得使用简单密码、生日、连续数字 | ❌ `123456`、❌ `admin` |

#### 2.2.2 账号使用守则

✅ **应该做的**：
- 妥善保管账号密码，不向他人透露
- 离开工位时锁定屏幕或退出系统
- 发现异常登录立即联系管理员
- 定期更新密码

❌ **禁止行为**：
- 将账号借给他人使用
- 使用公共电脑登录后不清除记录
- 在多个系统使用相同密码
- 将密码记录在易被他人看到的地方

### 3.2 项目管理

#### 3.2.1 项目生命周期

```
┌──────────┐    ┌──────────┐    ┌──────────┐    ┌──────────┐    ┌──────────┐
│  立项    │ → │  规划    │ → │  执行    │ → │  监控    │ → │  结项    │
│          │    │          │    │          │    │          │    │          │
│ 创建项目  │    │ 设定里程碑│    │ 分配任务 │    │ 跟踪进度 │    │ 归档总结 │

└──────────┘    └──────────┘    └──────────┘    └──────────┘    └──────────┘
```

#### 3.2.2 创建新项目

**操作步骤**：
1. 点击左侧菜单「项目管理」
2. 点击页面右上角「新建项目」按钮
3. 填写项目基本信息：
   - 项目名称（必填）
   - 项目编号（自动生成或手动输入）
   - 项目类型（内部项目/外部项目）
   - 项目经理（必填）
   - 计划起止时间
   - 项目金额（合同金额、外采金额）
   - 项目描述
4. 点击「保存」创建项目

> 💡 **提示**：项目创建后，可在项目详情中继续添加里程碑、任务、供应商等信息。

#### 3.2.3 项目详情管理

项目详情页包含以下标签页：

| 标签页 | 功能说明 | 操作权限 |
|-------|---------|---------|
| **概览** | 项目基本信息、进度统计 | 所有成员 |
| **里程碑** | 关键节点管理、进度跟踪 | 项目经理+ |
| **任务** | 项目任务列表、分配情况 | 所有成员 |
| **供应商** | 关联供应商、合作记录 | 项目经理+ |
| **风险** | 风险登记、评估、应对 | 项目经理+ |
| **动态** | 项目操作日志、变更记录 | 所有成员 |

### 3.3 任务中心

#### 3.3.1 任务状态流转


#### 3.3.2 任务操作指南

**创建任务**：
1. 进入「任务中心」
2. 点击「新建任务」
3. 填写任务信息：
   - 任务标题（简洁明了）
   - 所属项目（选择关联项目）
   - 任务描述（详细说明）
   - 负责人（可多人）
   - 优先级（高/中/低）
   - 截止日期
4. 点击「保存」

1. 在任务列表或看板中找到任务
2. 点击任务进入详情
3. 更改状态下拉框
4. 填写进度百分比（进行中状态）
5. 添加状态变更说明（可选）
6. 点击「保存」

**批量操作**：
- 在列表视图中勾选多个任务
- 使用顶部工具栏进行批量操作：
  - 批量分配负责人
  - 批量修改状态
  - 批量设置优先级

|-----|------|------|

| **进行中** | 已开始执行 | 负责人手动更新 |

### 3.5 风险管理

#### 3.5.1 风险等级定义

```
                    影响程度
           低          中          高
         ┌────┬────┬────┬────┬────┐
    高   │ 中 │ 高 │极高│极高│极高│
概       ├────┼────┼────┼────┼────┤
率  中   │ 低 │ 中 │ 高 │极高│极高│
       ├────┼────┼────┼────┼────┤
    低   │ 低 │ 低 │ 中 │ 高 │ 高 │
         └────┴────┴────┴────┴────┘
```

| 等级 | 颜色标识 | 响应要求 |
|-----|---------|---------|
| **极高** | 🔴 红色 | 立即上报，启动应急预案 |
| **高** | 🟠 橙色 | 24小时内制定应对方案 |
| **中** | 🟡 黄色 | 一周内制定应对措施 |
| **低** | 🟢 绿色 | 定期监控，纳入观察 |

#### 3.5.2 风险登记流程

1. **识别风险**：发现潜在问题或已发生问题
2. **评估风险**：确定概率和影响程度
3. **登记风险**：在系统中创建风险记录
4. **制定对策**：确定应对策略和责任人
5. **跟踪监控**：定期更新风险状态
6. **风险关闭**：风险消除或转化为问题

## 四、常见问题解答

### 4.1 登录相关问题

#### Q1: 忘记密码怎么办？

**解决方案**：
1. 在登录页面点击「忘记密码」
2. 输入注册邮箱或用户名
3. 查收密码重置邮件
4. 点击邮件中的重置链接
5. 设置新密码

> ⚠️ 如果无法自助重置，请联系系统管理员协助处理。

#### Q2: 账号被锁定怎么办？

**可能原因**：
- 连续多次输入错误密码
- 长时间未登录被系统冻结
- 管理员手动锁定

**解决方案**：
1. 等待 30 分钟后自动解锁
2. 或联系管理员手动解锁

#### Q3: 提示"账号不存在"？

**排查步骤**：
1. 确认输入的是用户名而非邮箱
2. 检查是否有多余空格
3. 确认大小写是否正确
4. 联系管理员确认账号状态

### 4.2 功能使用问题

#### Q4: 为什么看不到某些功能菜单？

**可能原因**：
- 权限不足：该功能需要特定角色权限
- 功能未启用：部分功能可能被管理员禁用
- 浏览器缓存：尝试清除缓存后重新登录

**解决方案**：
1. 确认自己的角色权限
2. 联系管理员申请权限
3. 清除浏览器缓存（Ctrl+Shift+Delete）

#### Q5: 任务状态无法更新？

**排查步骤**：
1. 确认自己是任务的负责人或创建者
2. 检查任务是否已被归档
3. 确认网络连接正常
4. 尝试刷新页面后重试

#### Q6: 无法创建项目？

**可能原因**：
- 权限不足：只有项目经理及以上角色可创建项目
- 必填信息未填写完整

**解决方案**：
1. 联系管理员确认角色权限
2. 检查表单必填项是否已填写

### 4.3 数据问题

#### Q7: 统计数据不准确？

**可能原因**：
1. 数据有延迟（部分统计每小时更新）
2. 筛选条件设置不正确
3. 权限限制导致部分数据不可见

**建议**：
- 等待几分钟后刷新
- 检查筛选条件
- 联系管理员确认数据权限

### 4.4 系统性能问题

#### Q8: 页面加载缓慢？

**优化建议**：
1. 检查网络连接
2. 清除浏览器缓存
3. 关闭不必要的浏览器标签页
4. 使用 Chrome 或 Edge 浏览器
5. 避免在高峰时段操作大数据量

#### Q9: 图表显示异常？

**解决方案**：
1. 刷新页面
2. 检查浏览器缩放比例（建议 100%）
3. 尝试切换浏览器
4. 清除浏览器缓存

### 4.5 移动端问题

#### Q10: 手机访问功能不全？

**说明**：移动端目前支持查看和部分操作，复杂功能建议在电脑端使用。

**支持的移动端功能**：
- 查看项目和任务
- 更新任务状态
- 接收通知

## 五、违规处理机制

### 5.1 违规行为定义

#### 5.1.1 严重违规（立即处理）

| 违规行为 | 具体表现 | 处理措施 |
|---------|---------|---------|
| **账号共享** | 将账号密码告知他人 | 警告+强制改密，再犯封禁 |
| **数据泄露** | 导出敏感数据外传 | 立即封禁，追究责任 |
| **恶意操作** | 故意删除、篡改他人数据 | 立即封禁，追究责任 |
| **越权访问** | 利用漏洞访问未授权数据 | 立即封禁，技术调查 |

#### 5.1.2 一般违规（警告处理）

| 违规行为 | 具体表现 | 处理措施 |
|---------|---------|---------|
| **弱密码** | 使用简单密码 | 强制修改密码 |
| **长期不更新密码** | 超过 180 天未改密 | 提醒，限期修改 |
| **不规范操作** | 不按要求填写信息 | 提醒，培训 |
| **资源滥用** | 大量下载、频繁刷新 | 警告，限制操作频率 |

#### 5.1.3 轻微违规（提醒改进）

| 违规行为 | 具体表现 | 处理措施 |
|---------|---------|---------|
| **信息不完整** | 个人资料缺失 | 提醒完善 |
| **未及时更新状态** | 任务完成后不标记 | 提醒规范操作 |

### 5.2 违规处理流程

```
┌──────────┐    ┌──────────┐    ┌──────────┐    ┌──────────┐
│ 发现违规  │ → │ 核实确认  │ → │ 分级处理  │ → │ 记录归档  │
└──────────┘    └──────────┘    └──────────┘    └──────────┘
     │                              │
     ▼                              ▼
┌──────────┐                  ┌──────────┐
│ 系统监测  │                  │ 严重违规  │
│ 用户举报  │                  │ 一般违规  │
│ 审计发现  │                  │ 轻微违规  │
└──────────┘                  └──────────┘
```

### 5.3 处罚等级

| 等级 | 处罚措施 | 适用情形 |
|-----|---------|---------|
| **提醒** | 系统消息提醒 | 首次轻微违规 |
| **警告** | 书面警告，限期整改 | 重复轻微违规或一般违规 |
| **限制** | 限制部分功能使用 | 多次违规或较严重违规 |
| **封禁** | 临时或永久封禁账号 | 严重违规 |

### 5.4 申诉流程

**申诉条件**：
- 对处罚决定有异议
- 能提供证据证明无违规
- 在处罚通知后 7 个工作日内提出

**申诉步骤**：
1. 向直属主管提交书面申诉
2. 主管审核后转交系统管理员
3. 管理员 3 个工作日内复核
4. 通知申诉结果
