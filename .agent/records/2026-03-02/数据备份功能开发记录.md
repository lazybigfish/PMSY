# 数据备份与迁移功能开发记录

## 功能概述

为 PMSY 项目管理系统实现完整的数据备份与恢复功能，支持一键备份、定时备份、数据恢复等能力。

## 开发内容

### 1. 数据库脚本

#### 1.1 迁移脚本（用于现有数据库升级）
- **040_create_backup_records.sql**: 创建备份记录表
- **041_add_backup_schedule_config.sql**: 添加定时备份配置到系统设置

#### 1.2 初始化脚本（用于新数据库初始化）
- **022_create_backup_records.sql**: 创建备份记录表
  - 字段：id, name, description, file_path, file_size, manifest, status, error_message, created_by, created_at, completed_at
  - 索引：status, created_by, created_at

- **023_add_backup_schedule_config.sql**: 添加定时备份配置到系统设置
  - backup_schedule: 定时备份配置（启用状态、cron表达式、时区、保留数量、备份选项）
  - backup_storage: 备份存储配置（路径、最大文件大小、允许扩展名）

### 2. 后端核心服务

#### 2.1 BackupService.ts
- 创建备份：导出数据库数据、收集文件、打包ZIP、保存到存储目录
- 获取备份列表和详情
- 删除备份
- 应用保留策略（自动保留最近10个备份）
- 并发控制（防止同时执行多个备份）

#### 2.2 RestoreService.ts
- 预览备份包：验证manifest、检测数据冲突、生成预览信息
- 执行恢复：全量覆盖/增量合并两种模式
- 冲突处理：跳过/覆盖/重命名
- 事务支持：确保数据一致性

#### 2.3 ManifestValidator.ts
- 验证备份清单格式
- 版本兼容性检查
- 必需表验证

#### 2.4 VersionChecker.ts
- 版本号比较
- 数据库版本兼容性检查
- 应用版本兼容性检查

### 3. 后端 API 路由

| 接口 | 方法 | 功能 |
|------|------|------|
| /api/system/backup | POST | 创建备份 |
| /api/system/backup | GET | 获取备份列表 |
| /api/system/backup/:id | GET | 获取备份详情 |
| /api/system/backup/:id/download | GET | 下载备份文件 |
| /api/system/backup/:id | DELETE | 删除备份 |
| /api/system/backup/:id/verify | POST | 验证备份完整性 |
| /api/system/restore/preview | POST | 预览备份包 |
| /api/system/restore | POST | 执行数据恢复 |
| /api/system/backup/schedule | GET | 获取定时备份配置 |
| /api/system/backup/schedule | PUT | 更新定时备份配置 |

### 4. 定时备份调度器

- 使用 node-cron 实现定时任务
- 默认每天凌晨 2:00 执行
- 支持自定义 cron 表达式
- 自动应用保留策略

- 整合三个标签页（备份管理、数据恢复、定时备份）
- 状态管理和数据流

## 使用说明

### 创建备份
1. 进入「系统设置」→「数据备份」→「备份管理」
2. 点击「创建备份」按钮
3. 填写备份名称和描述
4. 选择备份选项（可选）
5. 点击「开始备份」

### 下载备份
1. 在备份列表中找到已完成的备份
2. 点击下载按钮
3. 备份文件将下载到本地

### 恢复数据
1. 进入「数据备份」→「数据恢复」
2. 上传备份文件（ZIP格式）
3. 查看预览信息
4. 选择恢复模式
5. 点击「开始恢复」并确认

### 设置定时备份
1. 进入「数据备份」→「定时备份」
2. 启用定时备份开关
3. 选择备份时间
4. 设置保留数量
5. 点击「保存设置」

## 注意事项

1. **权限控制**：仅管理员可访问备份恢复功能
2. **存储空间**：确保服务器有足够的磁盘空间存储备份
3. **备份时间**：建议选择系统负载较低的时间进行备份
4. **恢复风险**：全量覆盖将清空现有数据，请谨慎操作
5. **版本兼容**：备份包包含版本信息，不支持向后兼容
