# 端口配置修改记录

## 修改原因

开发环境的 80 端口另有用途，需要将业务使用端口从 80 改为 6969。

## 修改时间

2026-03-02

## 修改内容

### 1. 核心配置文件

#### config/docker/docker-compose.yml
- **修改内容**: 将 nginx 服务的端口映射从 `"80:80"` 改为 `"6969:80"`
- **影响**: Docker 容器启动后，主机的 6969 端口会映射到容器的 80 端口

#### config/nginx/nginx.conf
- **修改内容**: 将 server 的 listen 端口从 `80` 改为 `6969`
- **影响**: Nginx 服务会监听 6969 端口

### 2. 部署脚本访问地址提示

#### deploy/fresh-install/deploy.sh
- **修改内容**: 部署完成后的访问地址提示添加端口 `:6969`
- **修改前**: `http://$DEPLOY_SERVER_IP`
- **修改后**: `http://$DEPLOY_SERVER_IP:6969`

#### deploy/fresh-install/deploy-v2.sh
- **修改内容**: 部署完成后的访问地址提示添加端口 `:6969`
- **修改前**: `http://$DEPLOY_SERVER_IP`
- **修改后**: `http://$DEPLOY_SERVER_IP:6969`

#### deploy/update/deploy.sh
- **修改内容**: 部署完成后的访问地址提示添加端口 `:6969`
- **修改前**: `http://$SERVER_IP`
- **修改后**: `http://$SERVER_IP:6969`

#### deploy/windows/fresh-install/deploy.ps1
- **修改内容**: 部署完成后的访问地址提示添加端口 `:6969`
- **修改前**: `http://$ServerIp`
- **修改后**: `http://$ServerIp:6969`

#### deploy/windows/update/deploy.ps1
- **修改内容**: 部署完成后的访问地址提示添加端口 `:6969`
- **修改前**: `http://$ServerIp`
- **修改后**: `http://$ServerIp:6969`

## 访问方式变更

### 修改前
```
前端: http://<服务器IP>
API: http://<服务器IP>/api/health
```

### 修改后
```
前端: http://<服务器IP>:6969
API: http://<服务器IP>:6969/api/health
```

## 注意事项

1. **防火墙配置**: 确保服务器的 6969 端口已开放
2. **现有部署**: 已部署的环境需要手动修改 docker-compose.yml 和 nginx.conf 后重启服务
3. **MinIO 管理界面**: MinIO 控制台端口 9001 保持不变，访问地址仍为 `http://<服务器IP>:9001`

### 3. 前端环境配置文件

#### config/env/.env.production
- **修改内容**: 将 `API_URL` 和 `VITE_API_URL` 添加端口 `:6969`
- **修改前**:
  ```env
  API_URL=http://43.136.69.250
  VITE_API_URL=http://43.136.69.250/api
  ```
- **修改后**:
  ```env
  API_URL=http://43.136.69.250:6969
  VITE_API_URL=http://43.136.69.250:6969/api
  ```

### 4. 前端 API 默认端口统一

#### src/lib/api.ts
- **修改内容**: 将默认端口从 `3000` 改为 `3001`
- **修改前**: `const API_BASE_URL = import.meta.env.VITE_API_URL || 'http://localhost:3000';`
- **修改后**: `const API_BASE_URL = import.meta.env.VITE_API_URL || 'http://localhost:3001';`

#### src/services/fileService.ts
- **修改内容**: 将两处默认端口从 `3000` 改为 `3001`
- **修改前**:
  - `const API_BASE_URL = import.meta.env.VITE_API_URL || 'http://localhost:3000';`
- **修改后**:
  - `const API_BASE_URL = import.meta.env.VITE_API_URL || 'http://localhost:3001';`

#### src/services/backupApi.ts
- **修改内容**: 将默认端口从 `3000` 改为 `3001`
- **修改前**: `const API_BASE_URL = import.meta.env.VITE_API_URL || 'http://localhost:3000';`
- **修改后**: `const API_BASE_URL = import.meta.env.VITE_API_URL || 'http://localhost:3001';`
