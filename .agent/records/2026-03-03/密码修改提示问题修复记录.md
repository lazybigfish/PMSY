# 密码修改提示问题修复记录

## 问题描述

服务器环境个人信息修改密码，实际上修改成功，但是提示修改失败。

## 问题分析

### 代码审查

1. **前端代码**（`src/pages/Profile.tsx`）：
   - 调用 `api.auth.updatePassword()` 修改密码
   - 成功时显示 "密码修改成功，请使用新密码登录"
   - 失败时显示 "密码修改失败: " + 错误信息

2. **后端代码**（`api-new/src/routes/auth.ts`）：
   - 调用 `updatePassword()` 服务函数更新密码
   - 成功时返回 `{ success: true, message: '密码更新成功' }`
   - 失败时通过 `next(error)` 传递错误

3. **服务层代码**（`api-new/src/services/authService.ts`）：
   - 验证旧密码
   - 哈希新密码
   - 更新数据库

### 可能原因

用户反馈"实际上修改成功，但是提示修改失败"，这意味着：
1. 密码确实被修改了（数据库更新成功）
2. 但前端显示了错误提示

最可能的原因是：**后端返回的响应格式有问题，或者响应没有正确发送**。

## 问题根因

通过检查 Nginx 日志发现：
```
POST /api/auth/v1/user/password HTTP/1.1" 499 0
```

**HTTP 499** 是 Nginx 特有的状态码，表示**客户端在服务器返回响应前关闭了连接**。

根本原因是：**用户点击弹窗外部（背景）时，弹窗关闭并取消了正在进行的 HTTP 请求**。

论坛帖子中也反馈了这个问题：
> "修改密码有点反人类，点弹框外的地方自动关闭弹框了，容易误点。建议弹框设置成手动关闭。"

## 修复内容

### 1. 后端修复（`api-new/src/routes/auth.ts`）

添加了日志记录和确保响应正确发送：

```typescript
router.post('/user/password', requireAuth, async (req: Request, res: Response, next: NextFunction) => {
  try {
    // ... 参数验证 ...

    await updatePassword(userId, old_password, new_password);

    console.log(`[Auth] Password updated successfully for user: ${userId}`);

    // 确保响应正确发送
    res.status(200).json({
      success: true,
      message: '密码更新成功',
    });
    return;
  } catch (error) {
    console.error('[Auth] Password update error:', error);
    next(error);
  }
});
```

### 2. 前端修复（`src/components/Modal.tsx` 和 `src/pages/Profile.tsx`）

为 `ModalForm` 组件添加 `closeOnBackdropClick` 属性，在提交过程中禁止点击背景关闭弹窗：

**Modal.tsx：**
```typescript
interface ModalFormProps extends ModalProps {
  // ... 其他属性 ...
  closeOnBackdropClick?: boolean;  // 新增
}

export function ModalForm({
  // ... 其他属性 ...
  closeOnBackdropClick = true,  // 默认允许点击背景关闭
  // ...
}: ModalFormProps) {
  // ...
  return (
    <div
      // ...
      onClick={closeOnBackdropClick ? onClose : undefined}  // 根据条件决定是否关闭
    >
```

**Profile.tsx：**
```typescript
<ModalForm
  // ... 其他属性 ...
  closeOnBackdropClick={!changingPassword}  // 提交中禁止点击背景关闭
>
```
