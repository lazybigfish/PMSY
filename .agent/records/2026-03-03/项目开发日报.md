### 1. 论坛图片无法显示问题修复

**问题描述**：
服务器环境（106.227.19.2:6969）中，论坛帖子的图片无法显示，控制台报错404。

**错误信息**：
```
GET http://106.227.19.2:6969/storage/v1/object/public/files/forum/image_xxx.png 404 (Not Found)
```

**问题原因**：
在 `ForumTab.tsx` 中创建帖子时，如果图片是通过粘贴上传的（blob URL），代码会硬编码使用 `http://localhost:9000` 作为图片URL前缀：

```typescript
// 修复前的问题代码
if (url.startsWith('blob:')) {
  const fileName = img.id.replace('forum/', '');
  url = `http://localhost:9000/files/forum/${fileName}`;  // ❌ 硬编码本地地址
}
```

**修复方案**：
使用 `VITE_API_URL` 环境变量动态构建图片URL：

```typescript
// 修复后的代码
const API_BASE_URL = import.meta.env.VITE_API_URL || 'http://localhost:3001';
if (url.startsWith('blob:')) {
  const fileName = img.id.replace('forum/', '');
  url = `${API_BASE_URL}/storage/v1/object/public/files/forum/${fileName}`;  // ✅ 使用环境变量
}
```

### 2. 401 Unauthorized 错误分析

**问题描述**：
控制台报错 `GET http://106.227.19.2:6969/api/auth/v1/user 401 (Unauthorized)`

**分析结果**：
这是正常的认证行为。当用户未登录或token过期时，前端会尝试获取用户信息，后端返回401表示未授权。

**处理流程**：
1. 前端应用启动时，`AuthContextNew.tsx` 调用 `api.auth.getUser()`
2. 如果用户未登录（没有token或token过期），后端返回401
3. 前端捕获错误，清除本地存储的token，设置用户状态为null
4. `ProtectedRoute` 组件检测到无用户状态，重定向到登录页

**结论**：这不是bug，是正常的认证流程。

**问题描述**：
服务器环境中，回复帖子时粘贴图片无法正常显示，控制台报错404。

```

```

1. `API_URL=http://106.227.19.2`（缺少端口6969）→ 生成 `http://106.227.19.2/storage/...`
2. 配置为localhost → 生成 `http://localhost/storage/...`

**修复方案**：

1. **配置文件修正**：`config/env/.env.production` 中 `API_URL` 已正确配置为 `http://106.227.19.2:6969`

2. **部署配置指南更新**：更新了文档，明确标注必须包含端口6969：
   ```bash
   API_URL=http://106.227.19.2:6969  # 必须包含端口
   VITE_API_URL=http://106.227.19.2:6969/api  # 必须包含端口
   ```

**修改文件**：
- `部署配置指南.md` - 更新配置示例和说明

### 4. 更新部署后数据库连接问题修复

**问题描述**：
执行更新部署后，服务器登录报错500，API容器不断重启。

**错误信息**：
```
Database connection failed: error: password authentication failed for user "pmsy"
```

**问题原因**：
更新部署脚本复制了新的 `.env` 配置文件（包含新的数据库密码），但PostgreSQL数据库是用旧密码初始化的，导致密码不匹配。

**修复方案**：
在PostgreSQL中修改用户密码，使其与新的 `.env` 配置一致：

```bash
# 修改数据库用户密码
docker exec -e PGPASSWORD=e4+GpY5OMar7AA4kIhHl pmsy-postgres psql -U pmsy -d pmsy -c "ALTER USER pmsy WITH PASSWORD 'e4+GpY5OMar7AA4kIhHl';"

# 重启API容器
docker compose restart api
```

**状态**：✅ 已修复，API服务正常运行
