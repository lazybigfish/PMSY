# 任务状态与进度联动修复记录

## 问题描述

任务详情页中，当任务处于"待办"(todo)状态时，更新任务进度后，任务状态没有自动联动变化。

例如：
- 任务状态为"待办"，进度为 0%
- 用户更新进度为 30%
- 期望：任务状态自动变为"进行中"
- 实际：任务状态仍为"待办"

## 修复方案

### 联动规则

| 当前状态 | 更新后进度 | 联动后状态 |
|---------|-----------|-----------|
| 待办 (todo) | > 0% 且 < 100% | 进行中 (in_progress) |
| 待办 (todo) | = 100% | 已完成 (done) |
| 进行中 (in_progress) | = 100% | 已完成 (done) |

### 代码修改

**文件**: `src/pages/tasks/TaskDetailPage.tsx`

**函数**: `handleProgressUpdate`

**修改前**:
```typescript
// 3. 更新任务进度和状态
const updateData: any = { progress };
if (progress === 100 && task.status !== 'done') {
  updateData.status = 'done';
  updateData.completed_at = new Date().toISOString();
}
await taskService.updateTask(task.id, updateData);
```

**修改后**:
```typescript
// 3. 更新任务进度和状态（状态联动逻辑）
const updateData: any = { progress };

// 状态联动规则：
// - 待办(todo) + 进度 > 0% → 进行中(in_progress)
// - 进行中(in_progress) + 进度 = 100% → 已完成(done)
if (progress === 100 && task.status !== 'done') {
  updateData.status = 'done';
  updateData.completed_at = new Date().toISOString();
} else if (progress > 0 && progress < 100 && task.status === 'todo') {
  updateData.status = 'in_progress';
}

await taskService.updateTask(task.id, updateData);
```

## 测试验证

### 测试场景 1：待办 → 进行中
1. 创建一个状态为"待办"的任务
2. 点击"更新进度"按钮
3. 输入进度 30%，填写更新内容
4. 提交后验证：
   - 任务状态变为"进行中"
   - 进度显示为 30%

### 测试场景 2：待办 → 已完成
1. 创建一个状态为"待办"的任务

4. 提交后验证：
   - 任务状态变为"已完成"
   - 进度显示为 100%
   - 显示完成时间
