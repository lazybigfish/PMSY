# 任务附件上传与显示修复记录

## 日期
2026-03-04

## 问题描述

### 问题1：上传附件报错
在任务进度更新页面（TaskProgressUpdateModal）上传附件时，出现以下错误：
```
TypeError: buckets?.some is not a function
    at uploadAttachments (TaskProgressUpdateModal.tsx:143:39)
    at async handleSubmit (TaskProgressUpdateModal.tsx:198:35)
```

### 问题2：附件显示与删除功能缺失
上传后的附件在任务详情页虽然有显示区域，但：
1. 样式较为简陋
2. 缺少删除按钮
3. 没有文件大小格式化显示
4. 缺少下载按钮
5. **最近更新区域只显示附件数量，不显示附件列表**

## 原因分析

### 问题1原因
`api.storage.listBuckets()` 返回的是 `{ data: StorageBucket[] | null; error: any }`，但代码中 `buckets?.some` 假设 `buckets` 一定是数组。如果 API 返回的不是数组（比如对象或其他格式），`some` 方法就会报错。

### 问题2原因
1. TaskDetail 组件（弹窗版）虽然有附件显示区域，但功能不够完善
2. **TaskDetailPage 组件（独立页面版）的"最近更新"区域只显示附件数量，没有显示附件列表**

## 修复方案

### 修复1：TaskProgressUpdateModal.tsx

**修改位置**：第143行

**修改前**：
```typescript
const bucketExists = buckets?.some(b => b.name === 'task-attachments');
```

**修改后**：
```typescript
const bucketExists = Array.isArray(buckets) && buckets.some(b => b.name === 'task-attachments');
```

**修复说明**：
- 添加 `Array.isArray(buckets)` 检查，确保 `buckets` 是数组类型后再调用 `some` 方法
- 这样即使 API 返回非数组类型（如对象、null 等），也不会报错

## 数据关系说明

```
task_progress_logs (进度日志表)
    ├── id
    ├── task_id
    ├── user_id
    ├── progress
    ├── description
    └── created_at

    ↓ 一对多关系

task_progress_attachments (进度附件表)
    ├── id
    ├── progress_log_id  ← 关联到进度日志
    ├── file_name
    ├── file_url
    ├── file_type
    └── file_size
```

**注意**：历史记录（task_history）和进度日志（task_progress_logs）是两个独立的系统：
- 历史记录：记录任务字段的变更（状态、优先级、截止日期等）
- 进度日志：记录进度更新，可以包含附件

## 权限控制

附件删除权限（TaskDetail.tsx 弹窗版）：
- 任务负责人（owner）可以删除任何附件
- 附件上传者可以删除自己上传的附件
- 其他用户无法删除附件
