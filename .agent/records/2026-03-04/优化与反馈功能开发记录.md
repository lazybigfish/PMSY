# 优化与反馈功能开发记录

## 开发日期
2026-03-04

## 功能概述
在水漫金山模块中新增二级功能"优化与反馈"，与水底探宝同级。该功能用于收集用户对系统的反馈、需求建议和问题报告，所有用户都可以查看、评论和参与讨论，管理员可以处理反馈并更新开发状态。

## 实现内容

### 1. 数据库表结构
- **feedback 表**：存储反馈记录
  - 基本信息：id, title, content, type, images, status
  - 提交人信息：created_by, created_by_name, created_at
  - 处理信息：handled_by, handled_by_name, handled_at, handle_result, handle_remark
  - 开发进度：dev_status, dev_status_updated_at, dev_status_updated_by

- **feedback_comments 表**：存储评论
  - 评论内容：id, feedback_id, content
  - 评论人信息：created_by, created_by_name, created_by_avatar, created_at
  - 回复功能：reply_to, reply_to_name

- **feedback_status_history 表**：存储状态变更历史
  - 变更信息：id, feedback_id, from_status, to_status, remark
  - 操作人信息：created_by, created_by_name, created_at

### 2. 前端组件

#### 2.1 类型定义
- 文件：`src/types/feedback.ts`
- 包含：FeedbackType, FeedbackStatus, Feedback, FeedbackComment, FeedbackStatusHistory 等类型定义

#### 2.2 服务层
- 文件：`src/services/feedbackService.ts`
- 功能：获取反馈列表、获取反馈详情、创建反馈、处理反馈、更新状态、删除反馈、评论相关操作、获取状态历史

#### 2.3 Hooks
- 文件：`src/hooks/useFeedback.ts`
- 包含：useFeedbackList, useFeedbackStats, useFeedbackDetail, useCreateFeedback, useFeedbackComments, useFeedbackStatusHistory

#### 2.4 页面组件
- **FeedbackTab.tsx**：反馈列表页
  - 统计卡片（全部/待处理/进行中/已完成）
  - 状态筛选、类型筛选
  - 列表排序（进行中在前，已完成/拒绝在后）
  - 评论数量显示

- **FeedbackDetailPage.tsx**：反馈详情页
  - 反馈内容展示
  - 图片预览
  - 处理记录（支持状态历史）
  - 评论区域（支持回复）
  - 管理员操作（处理反馈、更新状态、删除）

#### 2.5 子组件
- **FeedbackSubmitModal.tsx**：提交反馈弹窗
- **FeedbackComment.tsx**：评论组件（支持楼中楼）

### 3. 后端 API 修改
- 文件：`api-new/src/routes/rest.ts`
- 修改内容：
  - POST 路由：自动添加 created_by_name 和 created_by_avatar
  - PATCH 路由：更新 feedback 状态时自动记录状态历史到 feedback_status_history 表

### 4. 路由配置
- 列表页：`/water?tab=feedback`
- 详情页：`/water/feedback/:id`

## 状态颜色定义
- 🔴 已拒绝（红色）
- 🟠 待处理/已接受/开发中/测试中（橙色）
- 🟢 待发版/已完成（绿色）

## 状态历史记录
每次管理员更新反馈状态时，系统会：
1. 更新 feedback 表的 status 字段
2. 在 feedback_status_history 表中插入一条记录，包含：
   - from_status：变更前的状态
   - to_status：变更后的状态
   - remark：备注
   - created_by_name：操作人
   - created_at：操作时间
