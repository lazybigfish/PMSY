# 优化与反馈功能需求文档

## 1. 功能概述

### 1.1 功能定位
在水漫金山模块中新增二级功能"优化与反馈"，与水底探宝同级。该功能用于收集用户对系统的反馈、需求建议和问题报告，所有用户都可以查看、评论和参与讨论。

### 1.2 目标用户
- **所有用户**：提交反馈、查看所有反馈、发表评论、参与讨论
- **管理员**：处理反馈（接受/拒绝）、更新开发状态

---

## 2. 功能模块设计

### 2.1 模块结构

```
水漫金山
├── 水底探宝（现有）
└── 优化与反馈（新增）
    ├── 反馈列表页
    ├── 反馈详情页（含评论）
    └── 提交反馈弹窗
```

### 2.2 路由规划

| 路由路径 | 页面组件 | 说明 |
|---------|---------|------|
| `/water?tab=feedback` | FeedbackTab | 反馈列表（Tab 切换） |
| `/water/feedback/:id` | FeedbackDetailPage | 反馈详情页 |

## 3. 数据模型设计

### 3.1 反馈记录表 (feedback)

```typescript
interface Feedback {
  id: string;                    // 主键 UUID
  title: string;                 // 反馈标题
  content: string;               // 反馈内容
  type: 'bug' | 'feature' | 'improvement' | 'other';  // 反馈类型
  images: string[];              // 图片URL数组
  status: FeedbackStatus;        // 处理状态
  
  // 提交人信息
  created_by: string;            // 提交人ID
  created_by_name: string;       // 提交人姓名
  created_at: string;            // 提交时间
  
  // 处理信息
  handled_by?: string;           // 处理人ID
  handled_by_name?: string;      // 处理人姓名
  handled_at?: string;           // 处理时间
  handle_result?: 'accepted' | 'rejected';  // 处理结果
  handle_remark?: string;        // 处理备注
  
  // 开发进度（接受后）
  dev_status?: 'developing' | 'testing' | 'ready';  // 开发状态
  dev_status_updated_at?: string;  // 状态更新时间
  dev_status_updated_by?: string;  // 状态更新人
  
  updated_at: string;            // 更新时间
}
```

### 3.2 反馈评论表 (feedback_comment)

```typescript
interface FeedbackComment {
  id: string;                    // 主键 UUID
  feedback_id: string;           // 关联反馈ID
  content: string;               // 评论内容
  
  // 评论人信息
  created_by: string;            // 评论人ID
  created_by_name: string;       // 评论人姓名
```

## 6. 接口设计

### 6.1 后端 API

| 接口 | 方法 | 说明 | 权限 |
|-----|------|------|------|
| `/api/feedback` | GET | 获取反馈列表 | 所有用户 |
| `/api/feedback/:id` | GET | 获取反馈详情 | 所有用户 |
| `/api/feedback` | POST | 提交新反馈 | 所有用户 |
| `/api/feedback/:id` | DELETE | 删除反馈 | 管理员 |
| `/api/feedback/:id/handle` | PUT | 处理反馈 | 管理员 |
| `/api/feedback/:id/status` | PUT | 更新开发状态 | 管理员 |
| `/api/feedback/:id/comments` | GET | 获取评论列表 | 所有用户 |
| `/api/feedback/:id/comments` | POST | 发表评论 | 所有用户 |
| `/api/feedback/comments/:id` | DELETE | 删除评论 | 管理员 |

### 6.2 请求/响应示例

**提交反馈**

```http
POST /api/feedback
Content-Type: application/json

{
  "title": "登录页面加载慢",
  "content": "在弱网环境下，登录页面加载需要5秒以上",
  "type": "bug",
  "images": ["https://.../image1.jpg"]
}
```

**发表评论**

```http
POST /api/feedback/123/comments
Content-Type: application/json

{
  "content": "我也遇到了同样的问题，希望能尽快修复",
  "reply_to": null  // 回复某条评论时填写评论ID
}
```

**处理反馈**

```http
PUT /api/feedback/123/handle
Content-Type: application/json

{
  "result": "accepted",  // 或 "rejected"
  "remark": "已评估，将在下个版本优化"
}
```

**更新状态**

```http
PUT /api/feedback/123/status
Content-Type: application/json

{
  "status": "developing",  // developing/testing/ready/completed
  "remark": "开始开发"
}
```

## 7. 数据库表结构

```sql
-- 反馈表
CREATE TABLE IF NOT EXISTS feedback (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    title VARCHAR(200) NOT NULL,
    content TEXT NOT NULL,
    type VARCHAR(20) NOT NULL CHECK (type IN ('bug', 'feature', 'improvement', 'other')),
    images JSONB DEFAULT '[]',
    status VARCHAR(20) NOT NULL DEFAULT 'pending' CHECK (status IN ('pending', 'accepted', 'rejected', 'developing', 'testing', 'ready', 'completed')),
    
    -- 提交人信息
    created_by UUID NOT NULL REFERENCES users(id),
    created_by_name VARCHAR(100) NOT NULL,
    created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
    
    -- 处理信息
    handled_by UUID REFERENCES users(id),
    handled_by_name VARCHAR(100),
    handled_at TIMESTAMP WITH TIME ZONE,
    handle_result VARCHAR(20) CHECK (handle_result IN ('accepted', 'rejected')),
    handle_remark TEXT,
    
    -- 开发进度
    dev_status VARCHAR(20) CHECK (dev_status IN ('developing', 'testing', 'ready')),
    dev_status_updated_at TIMESTAMP WITH TIME ZONE,
    dev_status_updated_by UUID REFERENCES users(id),
    
    updated_at TIMESTAMP WITH TIME ZONE DEFAULT NOW()
);

-- 反馈评论表
CREATE TABLE IF NOT EXISTS feedback_comment (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    feedback_id UUID NOT NULL REFERENCES feedback(id) ON DELETE CASCADE,
    content TEXT NOT NULL,
    
    -- 评论人信息
    created_by UUID NOT NULL REFERENCES users(id),
    created_by_name VARCHAR(100) NOT NULL,
    created_by_avatar VARCHAR(500),
    created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
    
    -- 回复功能
    reply_to UUID REFERENCES feedback_comment(id),
    reply_to_name VARCHAR(100)
);

-- 索引
CREATE INDEX idx_feedback_status ON feedback(status);
CREATE INDEX idx_feedback_created_by ON feedback(created_by);
CREATE INDEX idx_feedback_created_at ON feedback(created_at DESC);
CREATE INDEX idx_feedback_type ON feedback(type);

CREATE INDEX idx_feedback_comment_feedback_id ON feedback_comment(feedback_id);
CREATE INDEX idx_feedback_comment_created_at ON feedback_comment(created_at);
```

## 9. 交互流程

### 9.1 用户提交反馈流程

```
用户点击[提交反馈]
       ↓
打开提交弹窗
       ↓
填写类型、标题、描述、上传图片
       ↓
点击[提交]
       ↓
验证表单 → 调用API创建反馈
       ↓
关闭弹窗，刷新列表，显示成功提示
```

### 9.2 用户发表评论流程

```
用户进入反馈详情页
       ↓
在评论输入框填写内容
       ↓
点击[发表评论] 或 回复某条评论
       ↓
调用API创建评论
       ↓
刷新评论列表，显示新评论
```

### 9.3 管理员处理反馈流程

```
管理员查看反馈列表/详情
       ↓
点击[处理反馈]
       ↓
选择接受/拒绝
       ↓
如拒绝：填写拒绝原因 → 确认 → 反馈状态变为"已拒绝"
       ↓
如接受：反馈状态变为"已接受" → 可更新开发状态
       ↓
开发中 → 测试中 → 待发版 → 已完成
```

## 10. 非功能性需求

### 10.1 性能要求
- 列表页加载时间 < 1秒
- 图片上传支持压缩，单张最大 5MB
- 列表分页，每页 20 条
- 评论支持分页加载，每页 10 条

### 10.2 安全要求
- 图片上传校验格式（jpg/png/gif）
- 防止 XSS，内容需要转义
- 管理员权限校验

### 10.3 兼容性
- 支持移动端浏览
- 图片预览支持手势缩放

2. **数据统计**：反馈处理效率统计
3. **标签系统**：为反馈添加标签便于分类
4. **投票功能**：用户可以对反馈进行投票，优先处理高票需求
5. **富文本编辑**：支持 Markdown 或富文本编辑反馈内容
