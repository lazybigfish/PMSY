# 功能模块层级路径标识需求文档

## 需求背景

当前功能模块的排序存在问题：
1. 插入/导入后按字母排序，不符合用户习惯
2. 调整模块顺序后，任务详情页展示顺序不能同步更新
3. 缺乏直观的层级标识，无法一眼看出模块的层级关系

## 需求目标

实现功能模块的层级路径标识系统，满足以下要求：
1. **固定标识**：每个模块有唯一的层级路径标识（如 `1`、`1.1`、`2.1.1`）
2. **直观展示**：从标识可直观看出模块层级和顺序
3. **灵活调整**：支持模块移动、升降级，标识自动更新
4. **统一排序**：所有页面按路径标识排序展示

## 方案设计

### 数据库设计

在 `project_modules` 表新增 `path` 字段：

```sql
-- 新增 path 字段
ALTER TABLE project_modules ADD COLUMN path TEXT;

-- 创建索引加速排序查询
CREATE INDEX idx_project_modules_path ON project_modules(path);
```

### 字段职责

| 字段 | 作用 | 说明 |
|------|------|------|
| `path` | **核心标识**：`1.1`、`2.1.1` | 用于展示和排序，替代 sort_order |
| `parent_id` | 父子关联 | 构建树结构 |
| `level` | 层级深度 | 辅助字段，方便查询和样式计算 |
| `sort_order` | ~~废弃~~ | 被 path 完全替代 |

### 路径生成规则

```
顶级模块：1, 2, 3...（按插入顺序递增）
子模块：1.1, 1.2, 2.1, 2.1.1...（父路径 + . + 序号）
```

### 排序规则

```sql
ORDER BY path ASC
-- 结果：1, 1.1, 1.2, 2, 2.1, 2.1.1, 2.2...
```

## 功能范围

### 1. 数据库迁移
- 添加 `path` 字段
- 为现有数据生成 path
- 创建索引

### 2. 功能模块页面（FunctionalModules.tsx）
- 插入模块时生成 path
- 导入模块时按行顺序生成 path
- 移动模块时更新 path 及其子模块
- 按 path 排序展示
- 显示 path 标识

### 3. 任务详情页（TaskDetailPage.tsx / TaskDetail.tsx）
- 功能模块按 path 排序展示

### 4. 类型定义更新
- ProjectModule 接口添加 path 字段

## 示例

### 模块结构示例

```
1 系统管理
  1.1 用户管理
  1.2 角色管理
2 订单管理
  2.1 订单列表
    2.1.1 待处理订单
    2.1.2 已完成订单
  2.2 订单统计
```

### 移动示例

将 `2.1 订单列表` 移动到 `1 系统管理` 下：
```
1 系统管理
  1.1 用户管理
  1.2 角色管理
  1.3 订单列表      ← 原 2.1，path 更新为 1.3
    1.3.1 待处理订单  ← 原 2.1.1，path 更新为 1.3.1
    1.3.2 已完成订单  ← 原 2.1.2，path 更新为 1.3.2
2 订单管理
  2.1 订单统计      ← 原 2.2，path 更新为 2.1
```

## 验收标准

1. ✅ 新插入的模块有正确的 path
2. ✅ 导入的模块按 Excel 行顺序生成 path
3. ✅ 移动模块后 path 及其子模块 path 自动更新
4. ✅ 功能模块列表按 path 排序展示
5. ✅ 任务详情页功能模块按 path 排序展示
6. ✅ 页面展示 path 标识（如 `1.1 用户管理`）
