## 日期
2026-03-04

## 方案概述

采用**混合方案**（方案三），根据文件类型和 bucket 自动选择访问策略：

| 文件类型 | 访问策略 | 说明 |
|---------|---------|------|
| 公开图片（forum, images, avatars） | **直接访问 MinIO** | 本来就是公开的，性能好 |
| 私有文件（documents, contracts） | **后端代理** | 需要权限控制 |
| 临时文件 | **预签名 URL** | 有过期时间，更安全 |

---

## 实施内容

### 1. 后端文件访问服务

**文件**: `api-new/src/services/fileAccessService.ts`

功能：
- 支持三种访问策略：`DIRECT`、`PROXY`、`PRESIGNED`
- 根据 bucket 自动选择默认策略
- 提供公开/私有 bucket 管理

```typescript
// 公开 bucket 列表
const PUBLIC_BUCKETS = ['images', 'forum', 'avatars'];

// 私有 bucket 列表
const PRIVATE_BUCKETS = ['documents', 'contracts', 'private'];
```

### 2. 后端文件访问路由

**文件**: `api-new/src/routes/files.ts`

接口：
- `GET /api/files/:bucket/:path` - 后端代理访问（需要认证）
- `POST /api/files/presigned-urls` - 批量获取预签名 URL
- `GET /api/files/config` - 获取 MinIO 配置

### 3. 后端上传服务修改

**文件**: `api-new/src/services/storageService.ts`

修改：
- 使用 `fileAccessService.getDirectUrl()` 生成 URL
- 返回 MinIO 直链而不是后端代理 URL

### 4. 前端文件访问 Hook

**文件**: `src/hooks/useFileAccess.ts`

功能：
- 支持四种策略：`DIRECT`、`PROXY`、`PRESIGNED`、`AUTO`
- 自动根据 bucket 选择策略
- 缓存 MinIO 配置
- 批量获取 URL

使用示例：
```typescript
const { getUrl } = useFileAccess();

// 自动选择策略
const result = await getUrl({ bucket: 'forum', filePath: 'xxx.jpg' });
// result.url = http://localhost:9000/forum/xxx.jpg

// 强制使用代理
const result = await getUrl(
  { bucket: 'contracts', filePath: 'xxx.pdf' },
  { strategy: AccessStrategy.PROXY }
);
// result.url = http://localhost:3001/api/files/contracts/xxx.pdf
```

### 5. 前端组件修改

**文件**: `src/components/upload/ImageUploader.tsx`

修改：
- 移除 `crossOrigin="anonymous"` 属性（直接访问不需要）

## API 列表

### 上传文件
```
POST /api/upload
Content-Type: multipart/form-data

Body:
- file: File
- bucket: string
- folder?: string
- metadata?: object

Response:
{
  success: true,
  data: {
    id: string,
    fileName: string,
    fileUrl: string,  // MinIO 直链
    fileSize: number,
    mimeType: string,
    createdAt: string
  }
}
```

### 代理访问文件（需要认证）
```
GET /api/files/:bucket/:path
Authorization: Bearer <token>

Response: 文件流
```

### 批量获取预签名 URL
```
POST /api/files/presigned-urls
Authorization: Bearer <token>

Body:
{
  files: [{ bucket: string, filePath: string }],
  expirySeconds?: number
}

Response:
{
  success: true,
  data: [
    { bucket, filePath, url, success: true }
  ]
}
```

### 获取配置
```
GET /api/files/config

Response:
{
  success: true,
  data: {
    minioPublicUrl: string,
    publicBuckets: string[],
    privateBuckets: string[]
  }
}
```

## 使用方式

### 1. 上传图片（水漫金山）

```typescript
import { ImageUploader } from '@/components/upload';

<ImageUploader
  value={images}
  onChange={setImages}
  maxCount={9}
  bucket="forum"  // 公开 bucket，使用直接访问
  folder="posts"
  compress="post"
/>
```

上传后返回的 URL 示例：
```
http://localhost:9000/forum/1741061890123_abc123_image.jpg
```

### 2. 显示图片

```typescript
// 直接访问（公开图片）
<img src="http://localhost:9000/forum/xxx.jpg" />

// 后端代理（私有文件）
<img src="http://localhost:3001/api/files/contracts/xxx.pdf" />
```

### 3. 获取文件访问 URL（动态）

```typescript
import { useFileAccess, AccessStrategy } from '@/hooks/useFileAccess';

const { getUrl } = useFileAccess();

// 自动选择策略
const { url, strategy } = await getUrl({
  bucket: 'forum',
  filePath: 'xxx.jpg'
});
// strategy = 'direct'
// url = 'http://localhost:9000/forum/xxx.jpg'

// 强制使用预签名 URL
const { url, expiresAt } = await getUrl(
  { bucket: 'documents', filePath: 'xxx.pdf' },
  { strategy: AccessStrategy.PRESIGNED, expirySeconds: 3600 }
);
// url = 'http://localhost:9000/documents/xxx.pdf?X-Amz-Algorithm=...'
// expiresAt = 1741065490123
```

## 安全考虑

1. **公开 bucket** - 直接访问，但文件名包含随机 UUID，难以猜测
2. **私有 bucket** - 必须通过后端代理，需要认证
3. **预签名 URL** - 有过期时间，即使泄露也不会长期有效
4. **后端代理** - 可以添加审计日志、权限检查
