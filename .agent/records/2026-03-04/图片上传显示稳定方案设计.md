# 图片上传显示稳定方案设计

## 日期
2026-03-04

---

## 一、问题总结

### 1.1 当前架构问题

```
当前流程（复杂）：
┌─────────┐    POST /api/upload    ┌─────────┐    MinIO    ┌─────────┐
│  前端   │ ─────────────────────→ │  后端   │ ──────────→ │  MinIO  │
│         │                        │         │             │         │
│         │ ←───────────────────── │         │ ←────────── │         │
└─────────┘   返回代理URL           └─────────┘             └─────────┘
      │
      │ GET /storage/v1/object/public/...
      ↓
┌─────────┐
│  后端   │ （再次代理）
│  代理   │
└─────────┘
      │
      ↓ 获取文件
┌─────────┐
│  MinIO  │
└─────────┘
```

**问题：**
1. 文件访问需要经过两次后端代理，效率低
2. CORS 配置复杂，容易出错
3. 服务器负担重（文件流都经过后端）

### 1.2 根本原因

- 后端返回的 URL 是 `/storage/v1/object/public/...`，需要后端代理
- 前端图片加载时设置了 `crossOrigin="anonymous"`，触发 CORS 检查
- 后端 storage 路由虽然设置了 CORS 头，但可能配置不完整

---

## 二、推荐方案：直接访问 MinIO

### 2.1 新架构设计

```
新流程（简洁）：
┌─────────┐    POST /api/upload    ┌─────────┐    MinIO    ┌─────────┐
│  前端   │ ─────────────────────→ │  后端   │ ──────────→ │  MinIO  │
│         │                        │         │   存储文件   │         │
│         │ ←───────────────────── │         │             │         │
└─────────┘   返回直接访问URL        └─────────┘             └─────────┘
      │
      │ GET http://localhost:9000/files/forum/xxx.jpg
      ↓
┌─────────┐
│  MinIO  │ （直接访问，无需后端代理）
│  直链   │
└─────────┘
```

### 2.2 方案优势

1. **简单高效** - 文件访问不经过后端，减少服务器负担
2. **无 CORS 问题** - MinIO 配置 CORS 后，前端直接访问
3. **可扩展** - 支持 CDN、OSS 等外部存储
4. **统一接口** - 后端只负责上传和权限控制

## 三、具体实施步骤

### 步骤1：配置 MinIO CORS

在 MinIO 控制台或使用 mc 命令配置 CORS：

```bash
# 使用 mc 命令行工具
mc alias set local http://localhost:9000 minioadmin minioadmin
mc admin config set local cors \
  'expose_headers="*"' \
  'allow_origins="http://localhost:5173,http://localhost:3000"' \
  'allow_methods="GET,PUT,POST,DELETE"'
```

或在代码中配置（storageService.ts）：

```typescript
// 创建存储桶时设置 CORS
const corsConfig = {
  Version: '2012-10-17',
  Statement: [
    {
      Effect: 'Allow',
      Principal: '*',
      Action: ['s3:GetObject'],
      Resource: [`arn:aws:s3:::${bucket}/*`],
    },
  ],
};
await minioClient.setBucketPolicy(bucket, JSON.stringify(corsConfig));
```

### 步骤2：修改后端返回的 URL

修改 `storageService.ts` 中的 `uploadFile` 函数：

```typescript
// 修改前：返回后端代理 URL
const url = `${API_URL}/storage/v1/object/public/${bucket}/${filePath}`;

// 修改后：返回 MinIO 直链 URL
const MINIO_PUBLIC_URL = process.env.MINIO_PUBLIC_URL || 'http://localhost:9000';
const url = `${MINIO_PUBLIC_URL}/${bucket}/${filePath}`;
```

### 步骤3：前端移除 crossOrigin 属性

修改 `ImageUploader.tsx` 中的图片显示：

```typescript
// 修改前
<img src={image.url} crossOrigin="anonymous" />

// 修改后（MinIO 已配置 CORS，不需要 crossOrigin）
<img src={image.url} />
```

### 步骤4：保留后端代理（可选）

如果需要权限控制，可以保留后端代理路由，但默认使用直链：

```typescript
// 数据库记录中同时保存两种 URL
const fileRecord = {
  url: `${MINIO_PUBLIC_URL}/${bucket}/${filePath}`,  // 直链（默认使用）
  proxy_url: `${API_URL}/storage/v1/object/public/${bucket}/${filePath}`,  // 代理（需要权限控制时使用）
};
```

## 七、注意事项

1. **MinIO CORS 配置** - 必须正确配置，否则前端无法直接访问
2. **环境变量** - 开发环境和生产环境的 `MINIO_PUBLIC_URL` 可能不同
3. **HTTPS** - 生产环境必须使用 HTTPS，否则会有混合内容问题
4. **权限控制** - 如果需要限制访问，使用后端代理路由
5. **备份策略** - MinIO 数据需要定期备份
