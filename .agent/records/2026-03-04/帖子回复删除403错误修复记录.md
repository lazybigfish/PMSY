# 帖子回复删除403错误修复记录

## 问题描述

用户在帖子详情页删除自己回复的内容时，出现 403 Forbidden 错误，提示"该表未配置权限，仅管理员可访问"。

## 错误信息

```
api.ts:656   POST http://localhost:3001/rest/v1/delete 403 (Forbidden)
ForumPostDetailPage.tsx:422  Error deleting reply: Error: 该表未配置权限，仅管理员可访问
```

## 问题分析

1. 前端调用 `POST /rest/v1/delete` 接口删除回复
2. 后端在 `permissionService.ts` 中检查表权限
3. `tablePermissions` 对象中缺少 `forum_replies` 表的权限配置
4. 当表未配置权限时，系统默认只允许管理员访问

## 修复方案

在 `api-new/src/services/permissionService.ts` 的 `tablePermissions` 对象中添加 `forum_replies` 表的权限配置：

```typescript
forum_replies: {
  adminRoles: ['admin'],
  ownerField: 'author_id',
},
```

配置说明：
- `adminRoles: ['admin']` - 管理员拥有所有权限
- `ownerField: 'author_id'` - 作者（回复创建者）可以操作自己的回复

## 修改文件

- `api-new/src/services/permissionService.ts`

## 修复代码

```typescript
// 在 tablePermissions 对象中添加
forum_replies: {

```

## 验证结果

- 后端服务自动重启（热重载）
- 普通用户现在可以删除自己发布的回复
- 管理员可以删除所有回复
