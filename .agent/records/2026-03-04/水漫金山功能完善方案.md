| **分页功能** | 没有分页，一次性加载所有 | 数据量大时性能差 |
| **用户互动** | 缺少收藏功能 | 无法保存感兴趣的帖子 |
| **内容管理** | 缺少举报功能 | 不良内容无法处理 |
| **数据统计** | 缺少用户发帖统计 | 无法识别活跃用户 |

## 二、功能完善方案

### 2.1 核心功能（必须实现）

#### 2.1.1 帖子删除功能完善

**问题**：详情页缺少删除入口

**解决方案**：
- 在帖子详情页添加删除按钮（仅作者和管理员可见）
- 删除前二次确认
- 删除后返回列表页

#### 2.1.2 帖子编辑功能

**问题**：发布后无法修改

**解决方案**：
- 在帖子详情页添加编辑按钮（仅作者和管理员可见）
- 点击后弹出编辑弹窗
- 支持修改标题、内容、分类
- 记录编辑时间

#### 2.1.3 管理员功能扩展

**问题**：置顶/精华只能在列表页操作

**解决方案**：
- 在帖子详情页添加管理操作栏
- 支持置顶/取消置顶
- 支持设精华/取消精华
- 支持删除帖子

#### 2.1.4 分页功能

**问题**：一次性加载所有帖子

**解决方案**：
- 列表页添加分页组件
- 默认每页 20 条
- 支持跳转到指定页

### 2.2 增强功能（建议实现）

#### 2.2.1 排序功能

**需求**：支持多种排序方式

**排序选项**：
- 最新发布（默认）
- 最后回复
- 最多浏览
- 最多点赞
- 精华帖子

#### 2.2.2 收藏功能

**需求**：用户可以收藏感兴趣的帖子

**功能点**：
- 帖子详情页添加收藏按钮
- 我的收藏列表页面
- 取消收藏

#### 2.2.3 用户主页

**需求**：查看用户发帖历史

**功能点**：
- 点击用户头像进入用户主页
- 显示用户基本信息
- 显示该用户的所有帖子
- 显示发帖统计

#### 2.2.4 消息通知

**需求**：回复提醒

**功能点**：
- 有人回复我的帖子时通知
- 有人回复我的回复时通知
- 通知中心页面
- 已读/未读标记

### 2.3 高级功能（可选实现）

#### 2.3.1 关注用户

- 关注/取消关注用户
- 关注用户的帖子动态

#### 2.3.2 内容举报

- 举报不良内容
- 管理员审核举报

#### 2.3.3 帖子标签

- 自定义标签
- 按标签筛选

## 三、数据库变更

### 3.1 新增收藏表

```sql
-- 用户收藏帖子表
CREATE TABLE IF NOT EXISTS forum_favorites (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    user_id UUID REFERENCES profiles(id) ON DELETE CASCADE NOT NULL,
    post_id UUID REFERENCES forum_posts(id) ON DELETE CASCADE NOT NULL,
    created_at TIMESTAMPTZ DEFAULT NOW() NOT NULL,
    UNIQUE(user_id, post_id)
);

CREATE INDEX IF NOT EXISTS idx_forum_favorites_user_id ON forum_favorites(user_id);
CREATE INDEX IF NOT EXISTS idx_forum_favorites_post_id ON forum_favorites(post_id);
```

### 3.2 新增通知表

```sql
-- 用户通知表
CREATE TABLE IF NOT EXISTS forum_notifications (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    user_id UUID REFERENCES profiles(id) ON DELETE CASCADE NOT NULL,
    type TEXT NOT NULL CHECK (type IN ('reply_post', 'reply_reply', 'like_post', 'system')),
    title TEXT NOT NULL,
    content TEXT,
    related_post_id UUID REFERENCES forum_posts(id) ON DELETE CASCADE,
    related_reply_id UUID REFERENCES forum_replies(id) ON DELETE CASCADE,
    is_read BOOLEAN DEFAULT false,
    created_at TIMESTAMPTZ DEFAULT NOW() NOT NULL
);

CREATE INDEX IF NOT EXISTS idx_forum_notifications_user_id ON forum_notifications(user_id);
CREATE INDEX IF NOT EXISTS idx_forum_notifications_is_read ON forum_notifications(is_read);
CREATE INDEX IF NOT EXISTS idx_forum_notifications_created_at ON forum_notifications(created_at);
```

### 3.3 新增关注表

```sql
-- 用户关注表
CREATE TABLE IF NOT EXISTS forum_follows (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    follower_id UUID REFERENCES profiles(id) ON DELETE CASCADE NOT NULL,
    following_id UUID REFERENCES profiles(id) ON DELETE CASCADE NOT NULL,
    created_at TIMESTAMPTZ DEFAULT NOW() NOT NULL,
    UNIQUE(follower_id, following_id)
);

CREATE INDEX IF NOT EXISTS idx_forum_follows_follower ON forum_follows(follower_id);
CREATE INDEX IF NOT EXISTS idx_forum_follows_following ON forum_follows(following_id);
```

## 四、API 接口规划

### 4.1 帖子相关

| 接口 | 方法 | 说明 |
|-----|------|------|
| `/api/forum/posts` | GET | 获取帖子列表（支持分页、排序、筛选） |
| `/api/forum/posts` | POST | 创建帖子 |
| `/api/forum/posts/:id` | GET | 获取帖子详情 |
| `/api/forum/posts/:id` | PUT | 更新帖子 |
| `/api/forum/posts/:id` | DELETE | 删除帖子 |
| `/api/forum/posts/:id/pin` | POST | 置顶/取消置顶 |
| `/api/forum/posts/:id/essence` | POST | 设精华/取消精华 |
| `/api/forum/posts/:id/like` | POST | 点赞/取消点赞 |
| `/api/forum/posts/:id/favorite` | POST | 收藏/取消收藏 |

### 4.2 回复相关

| 接口 | 方法 | 说明 |
|-----|------|------|
| `/api/forum/posts/:id/replies` | GET | 获取回复列表 |
| `/api/forum/posts/:id/replies` | POST | 添加回复 |
| `/api/forum/replies/:id` | PUT | 更新回复 |
| `/api/forum/replies/:id` | DELETE | 删除回复 |
| `/api/forum/replies/:id/like` | POST | 点赞回复 |

### 4.3 用户相关

| 接口 | 方法 | 说明 |
|-----|------|------|
| `/api/forum/users/:id/posts` | GET | 获取用户帖子 |
| `/api/forum/users/:id/follow` | POST | 关注/取消关注 |
| `/api/forum/users/:id/profile` | GET | 获取用户论坛资料 |

### 4.4 通知相关

| 接口 | 方法 | 说明 |
|-----|------|------|
| `/api/forum/notifications` | GET | 获取通知列表 |
| `/api/forum/notifications/:id/read` | POST | 标记已读 |
| `/api/forum/notifications/read-all` | POST | 全部已读 |
| `/api/forum/notifications/unread-count` | GET | 获取未读数量 |

### 4.5 收藏相关

| 接口 | 方法 | 说明 |
