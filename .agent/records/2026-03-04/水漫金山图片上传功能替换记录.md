# 水漫金山图片上传功能替换记录

## 日期
2026-03-04

---

## 替换内容

将 `ForumTab.tsx` 中的 `PostImageUploader` 组件替换为新的 `ImageUploader` 组件。

---

## 修改文件

### 1. ForumTab.tsx

**文件路径**: `src/pages/water/ForumTab.tsx`

#### 修改1: 导入语句
```typescript
// 修改前
import { PostImageUploader, PostImage } from './components/PostImageUploader';

// 修改后
import { ImageUploader } from '../../components/upload';
```

#### 修改2: 状态类型
```typescript
// 修改前
const [formImages, setFormImages] = useState<PostImage[]>([]);

// 修改后
const [formImages, setFormImages] = useState<string[]>([]);
```

#### 修改3: 图片URL处理
```typescript
// 修改前
// 过滤出已上传成功的图片，转换 blob URL 为可存储的 URL
const API_BASE_URL = import.meta.env.VITE_API_URL || 'http://localhost:3001';
const uploadedImages = formImages
  .filter(img => !img.uploading && !img.error)
  .map(img => {
    let url = img.url;
    if (url.startsWith('blob:')) {
      const fileName = img.id.replace('forum/', '');
      url = `${API_BASE_URL}/storage/v1/object/public/files/forum/${fileName}`;
    }
    return url;
  });

// 修改后
// 使用新架构上传的图片URL已经是可直接存储的URL
const uploadedImages = formImages;
```

#### 修改4: 组件使用
```typescript
// 修改前
<PostImageUploader
  images={formImages}
  onChange={setFormImages}
  maxImages={9}
/>

// 修改后
<ImageUploader
  value={formImages}
  onChange={setFormImages}
  maxCount={9}
  maxSize={5 * 1024 * 1024}
  compress="post"
  bucket="images"
  folder="forum"
/>
```

---

## 新组件特性

`ImageUploader` 相比 `PostImageUploader` 的改进：

1. **统一的上传核心** - 使用 `UploadCore` 处理上传逻辑
2. **图片压缩** - 自动压缩图片（post模式：1920x1920，质量0.85）
3. **更好的错误处理** - 统一的错误类型和提示
4. **事务保证** - 后端使用数据库事务，确保文件和记录一致性
5. **进度显示** - 上传进度百分比显示
6. **图片预览** - 点击可预览大图

---

## 后端API

新上传API：
- **地址**: `POST /api/upload`
- **功能**: 统一文件上传接口
- **特性**:
  - 自动创建数据库记录
  - 支持 metadata 关联（taskId, projectId等）
  - 事务保证原子性
  - 权限控制

## 注意事项

1. 新组件使用 `value` 和 `onChange` 而不是 `images` 和 `onChange`
2. `formImages` 现在是 `string[]`（URL数组）而不是 `PostImage[]`
3. 不再需要手动转换 blob URL，新组件返回的已经是可直接使用的 URL
4. 后端服务已重启，新API已生效
