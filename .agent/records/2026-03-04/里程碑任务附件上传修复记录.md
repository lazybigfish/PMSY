# 里程碑任务附件上传修复记录

## 问题描述

在项目管理详情的里程碑页面，上传附件时报错：

```
PATCH http://localhost:3001/rest/v1/milestone_tasks?eq.id=90bdee4d-e7d8-444b-97ce-71e875627dfc 500 (Internal Server Error)

[API] Update error to milestone_tasks: Error: update "milestone_tasks" set "output_documents" = $1, "updated_by" = $2, "updated_at" = $3 where "id" = $4 returning * - invalid input syntax for type json
```

## 问题分析

### 根本原因

后端 REST API 的 PATCH 路由在处理 `milestone_tasks` 表的 `output_documents` JSONB 字段时，没有进行特殊处理。

### 详细原因

1. **数据库定义**：`milestone_tasks` 表的 `output_documents` 字段定义为 `JSONB DEFAULT '[]'::jsonb`
2. **前端行为**：前端在 [MilestoneTaskList.tsx](file:///Users/liiiiins/Downloads/文稿%20-%20liiiiins%20的%20MacBook%20Pro/Mweb/PMSY/src/pages/projects/components/MilestoneTaskList.tsx#L65-L74) 中构建 `docs` 数组并直接传递给 update 方法
3. **后端问题**：在 `rest.ts` 的 POST 路由中（插入操作），有对 `output_documents` 进行 JSONB 处理，但在 PATCH 路由（更新操作）中缺少此处理

### 代码对比

**POST 路由（有处理）**：
```typescript
// 处理 JSONB 字段：确保它们是对象/数组而不是字符串
if (table === 'milestone_tasks' && processedItem.output_documents) {
  if (typeof processedItem.output_documents === 'string') {
    processedItem.output_documents = JSON.parse(processedItem.output_documents);
  }
  // ...
}
```

**PATCH 路由（无处理）**：
```typescript
// 直接传递数据，没有处理 JSONB 字段
const updateData = {
  ...data,
  ...(hasUpdatedBy && userId ? { updated_by: userId } : {}),
};
```

### 代码变更

```typescript
/**
 * 处理 JSONB 字段，确保它们是对象/数组而不是字符串
 */
function processJsonbFields(table: string, data: Record<string, any>): Record<string, any> {
  const processedData = { ...data };
  
  // 处理 milestone_tasks 表的 output_documents 字段
  if (table === 'milestone_tasks' && processedData.output_documents !== undefined) {
    console.log('[REST] PATCH output_documents type:', typeof processedData.output_documents);
    console.log('[REST] PATCH output_documents value:', JSON.stringify(processedData.output_documents));
    
    if (typeof processedData.output_documents === 'string') {
      try {
        processedData.output_documents = JSON.parse(processedData.output_documents);
      } catch (e) {
        console.warn('[REST] Failed to parse output_documents as JSON:', processedData.output_documents);
        processedData.output_documents = [];
      }
    }
    // 确保是数组格式
    if (!Array.isArray(processedData.output_documents)) {
      console.warn('[REST] output_documents is not array, converting to array');
      processedData.output_documents = [];
    }
  }
  
  return processedData;
}
```

在 PATCH 路由中使用：
```typescript
// 处理 JSONB 字段
let updateData = processJsonbFields(table, data);

// 添加更新者 ID（仅当表有 updated_by 字段时）
if (hasUpdatedBy && userId) {
  updateData.updated_by = userId;
}
```

## 第二次修复：下载文件变成 JSON

### 问题
上传成功后，下载文件变成了 JSON 文件。

### 原因
Knex.js 在处理 PostgreSQL JSONB 字段时需要特殊处理。当直接传递 JavaScript 对象时，Knex 会将其序列化为字符串，但这个格式可能不符合 PostgreSQL JSONB 的解析要求。

### 修复
在 [dbService.ts](file:///Users/liiiiins/Downloads/文稿%20-%20liiiiins%20的%20MacBook%20Pro/Mweb/PMSY/api-new/src/services/dbService.ts) 中添加 `processJsonbForKnex` 函数：

```typescript
/**
 * 处理 JSONB 字段，将对象转换为 JSON 字符串以便 Knex 正确处理
 */
function processJsonbForKnex(data: Record<string, any>, table: string): Record<string, any> {
  const processed = { ...data };
  
  if (table === 'milestone_tasks' && processed.output_documents !== undefined) {
    if (typeof processed.output_documents === 'object') {
      processed.output_documents = JSON.stringify(processed.output_documents);
    }
  }
  
  return processed;
}
```

```typescript
const handleDeleteDoc = async (task: MilestoneTask, docIndex: number) => {
  const doc = task.output_documents?.[docIndex];
  if (!doc) return;

  if (!confirm(`确定要删除附件"${doc.name}"吗？`)) return;

  try {
    // 如果有 URL，先从存储中删除文件
    if (doc.url) {
      const urlMatch = doc.url.match(/\/project-documents\/(.+?)(?:\?|$)/);
      if (urlMatch && urlMatch[1]) {
        const filePath = urlMatch[1];
        await api.storage.from('project-documents').remove([filePath]);
      }
    }

    // 从数据库中移除文档记录
    const docs = [...(task.output_documents || [])];
    docs[docIndex] = {
      ...docs[docIndex],
      url: undefined,
      uploaded_at: undefined,
      uploaded_by: undefined
    };

    await api.db
      .from('milestone_tasks')
      .update({ output_documents: docs })
      .eq('id', task.id);

    fetchMilestoneTasks(task.milestone_id);
    alert('附件已删除');
  } catch (error) {
    console.error('Error deleting attachment:', error);
    alert('删除附件失败');
  }
};
```
