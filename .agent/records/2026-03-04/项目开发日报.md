## 技术方案要点

### path 生成规则
```
顶级模块：1, 2, 3...（按插入顺序递增）
子模块：1.1, 1.2, 2.1, 2.1.1...（父路径 + . + 序号）
```

### 排序逻辑
```typescript
// 按 path 字符串排序
ORDER BY path ASC
// 结果：1, 1.1, 1.2, 2, 2.1, 2.1.1, 2.2...
```

### 移动时 path 更新
- 同级移动：更新当前模块 path
- 跨级移动：更新当前模块及其所有子模块的 path
