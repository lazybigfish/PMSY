## 一、现有架构分析

### 1.1 数据库表结构（现有）

```
milestone_templates          # 里程碑模板表
├── id (UUID PK)
├── name (TEXT)
├── description (TEXT)
├── phase_order (INTEGER)
├── is_active (BOOLEAN)
├── version_id (UUID FK -> template_versions)
└── created_at/updated_at

milestone_task_templates     # 里程碑任务模板表
├── id (UUID PK)
├── milestone_template_id (UUID FK)
├── name (TEXT)
├── description (TEXT)
├── is_required (BOOLEAN)
├── output_documents (JSONB)
├── sort_order (INTEGER)
└── created_at/updated_at

template_versions            # 模板版本表
├── id (UUID PK)
├── name (TEXT)
├── version_number (TEXT)
├── description (TEXT)
├── is_active (BOOLEAN)
├── created_by (UUID FK)
└── created_at/updated_at

project_milestones           # 项目里程碑表
├── id (UUID PK)
├── project_id (UUID FK)
├── template_id (UUID FK)
├── name/description/status
├── phase_order/is_current
├── created_by (UUID FK)
└── created_at/updated_at

milestone_tasks              # 里程碑任务表
├── id (UUID PK)
├── milestone_id (UUID FK)
├── template_id (UUID FK)
├── name/description
├── is_required/is_completed/is_custom
├── output_documents (JSONB)
├── created_by/completed_by
└── created_at/updated_at
```

### 1.2 现有服务层

**projectService.ts**
- `getProjects()` - 获取项目列表
- `getProjectById()` - 获取项目详情（包含里程碑）
- `createProject()` - 创建项目（不初始化里程碑）
- `updateProject()` - 更新项目

**systemService.ts**
- `getMilestoneTemplates()` - 获取模板列表
- `getMilestoneTemplateWithTasks()` - 获取模板详情
- `createMilestoneTemplate()` - 创建模板
- `updateMilestoneTemplate()` - 更新模板
- `deleteMilestoneTemplate()` - 删除模板

**projectInitService.ts**（后端）
- `initializeProjectMilestones()` - 根据模板初始化项目里程碑
- `isProjectInitialized()` - 检查项目是否已初始化

### 1.3 现有页面组件

**前端页面**
- `Milestones.tsx` - 项目里程碑页面（标签页）
- `MilestoneTemplates.tsx` - 系统管理模板列表页
- `MilestoneSidebar.tsx` - 里程碑侧边栏

**文件**: `api-new/database/migrations/040_update_milestone_templates.sql`

```sql
-- 为 milestone_templates 表添加新字段
ALTER TABLE milestone_templates 
ADD COLUMN IF NOT EXISTS is_system BOOLEAN DEFAULT false,
ADD COLUMN IF NOT EXISTS is_public BOOLEAN DEFAULT false,
ADD COLUMN IF NOT EXISTS is_custom BOOLEAN DEFAULT false,
ADD COLUMN IF NOT EXISTS tags TEXT[] DEFAULT '{}',
ADD COLUMN IF NOT EXISTS created_by UUID REFERENCES profiles(id);

-- 为 template_versions 表添加新字段
ALTER TABLE template_versions
ADD COLUMN IF NOT EXISTS is_system BOOLEAN DEFAULT false,
ADD COLUMN IF NOT EXISTS is_public BOOLEAN DEFAULT false,
ADD COLUMN IF NOT EXISTS tags TEXT[] DEFAULT '{}',
ADD COLUMN IF NOT EXISTS use_count INTEGER DEFAULT 0;

-- 为 project_milestones 表添加初始化来源字段
ALTER TABLE project_milestones
ADD COLUMN IF NOT EXISTS init_source VARCHAR(50); -- 'template' | 'custom'

-- 更新现有数据：将现有模板标记为系统模板
UPDATE milestone_templates SET is_system = true, is_public = true WHERE is_system = false;
UPDATE template_versions SET is_system = true, is_public = true WHERE is_system = false;

-- 创建标签表
CREATE TABLE IF NOT EXISTS milestone_template_tags (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    name TEXT NOT NULL UNIQUE,
    color TEXT DEFAULT '#3B82F6',
    description TEXT,
    usage_count INTEGER DEFAULT 0,
    created_at TIMESTAMPTZ DEFAULT NOW() NOT NULL
);

-- 创建索引
CREATE INDEX IF NOT EXISTS idx_milestone_templates_is_system ON milestone_templates(is_system);
CREATE INDEX IF NOT EXISTS idx_milestone_templates_is_public ON milestone_templates(is_public);
CREATE INDEX IF NOT EXISTS idx_milestone_templates_created_by ON milestone_templates(created_by);
CREATE INDEX IF NOT EXISTS idx_milestone_templates_tags ON milestone_templates USING GIN(tags);
```

**验收标准**:

- [ ] 现有数据正确标记为系统模板

## 三、关键注意事项

### 3.1 与现有系统保持一致

1. **数据库操作**: 使用现有的 `db` 实例和事务处理模式
2. **API路由**: 遵循现有的 RESTful 路由规范
3. **前端服务**: 使用现有的 `api` 和 `apiClient`
4. **组件复用**: 优先使用现有组件（Modal、Button、Table等）
5. **类型定义**: 扩展现有类型，不重复定义

### 3.2 数据兼容性

1. **现有项目**: 已初始化的项目不受影响
2. **现有模板**: 自动标记为系统模板
3. **历史数据**: 迁移脚本幂等执行，可重复运行

### 3.3 权限控制

1. **普通用户**: 
   - 只能查看公开模板和自己的模板
   - 只能编辑/删除自己的模板
   
2. **管理员**:
   - 可以查看所有模板
   - 可以管理系统模板
   - 可以启用/禁用任何模板
   - 不能删除系统模板

### 3.4 避免重复造轮子

**复用清单**:
- ✅ `Modal` / `ModalForm` - 现有弹窗组件
- ✅ `Button` - 现有按钮组件
- ✅ `Table` / `DataTable` - 现有表格组件
- ✅ `Input` / `TextArea` - 现有表单组件
- ✅ `Select` / `Radio` / `Checkbox` - 现有选择组件
- ✅ `Tag` - 现有标签组件
- ✅ `Card` - 现有卡片组件
- ✅ `Pagination` - 现有分页组件
- ✅ `api` / `apiClient` - 现有API客户端
- ✅ `useToast` - 现有消息提示
