# 里程碑功能升级记录

## 2026-03-05 功能升级

### 1. 新增功能：重新初始化里程碑

#### 功能描述
在里程碑页面新增"重新初始化"按钮，允许用户：
1. 删除现有所有里程碑阶段、任务及相关附件
2. 重新选择模板或空白初始化
3. 使用事务确保数据一致性

#### 前端变更
- **文件**: `src/pages/projects/tabs/Milestones.tsx`
- **变更内容**:
  - 导入 `RefreshCw` 图标和 `TemplateSelectionModal` 组件
  - 添加 `showReinitModal` 状态控制弹窗显示
  - 实现 `handleReinitialize()` 函数：显示确认对话框后打开模板选择弹窗
  - 实现 `handleReinitConfirm()` 函数：调用后端 API 重新初始化
  - 在"保存为模板"按钮旁添加"重新初始化"按钮（橙色样式）
  - 添加 `TemplateSelectionModal` 组件用于选择模板

#### 后端变更
- **文件**: `api-new/src/routes/projects.ts`
- **新增 API**: `POST /api/projects/:projectId/milestones/reinitialize`
- **功能逻辑**:
  1. 开启数据库事务
  2. 检查项目存在性和用户权限（管理员或项目经理）
  3. 获取所有里程碑和任务 ID
  4. 删除 MinIO 中的附件文件
  5. 按依赖顺序删除数据库记录：
     - task_attachments（任务附件）
     - milestone_tasks（任务）
     - project_milestones（里程碑）
  6. 重新初始化里程碑：
     - 如传 templateId：从模板复制阶段和任务
     - 如未传 templateId：创建空白自定义里程碑
  7. 提交事务

#### 安全考虑
- 操作前需要用户二次确认
- 仅管理员或项目经理可执行
- 使用数据库事务确保原子性
- 删除附件时失败不影响整体流程（记录警告日志）
