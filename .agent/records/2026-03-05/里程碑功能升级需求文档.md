## 一、需求背景

当前项目管理系统中的里程碑功能存在以下问题：

1. **初始化机制僵化**：项目创建成功后，里程碑会自动根据系统默认模板初始化，缺乏灵活性
2. **模板选择受限**：用户无法在初始化时选择不同的模板方案
3. **个性化配置困难**：用户自定义的里程碑配置无法保存复用
4. **模板管理集中化不足**：缺乏对里程碑模板的统一管理界面

随着用户数量增长和项目类型多样化，需要一套更加灵活、可定制的里程碑管理系统。

### 1.1 现有默认模板保留说明

**现有的默认里程碑模板将完整保留，作为系统模板库的第一份公开系统模板。**

现有模板包含以下7个阶段：
1. **进场前阶段** - 5个任务（获取基础材料、明确干系人、组建项目团队、风险与预算分析、召开内部启动会）
2. **启动阶段** - 6个任务（编制基础文档、拆解建设内容、制定进度计划、召开项目启动会、筹备服务器资源、供应商/硬件下单）
3. **实施阶段** - 5个任务（需求调研、系统部署、第三方测评、培训与自查、监理核查）
4. **初验阶段** - 4个任务（整理验收文档、筹备并召开初验会、整改专家意见、上线试运行）
5. **试运行阶段** - 2个任务（试运行保障、项目结算与决算）
6. **终验阶段** - 3个任务（试运行总结、终验筹备与召开、终验整改）
7. **运维阶段** - 1个任务（项目移交）

该模板将作为：
- **系统模板**（is_system = true, is_public = true）
- **模板名称**：标准项目里程碑模板 v1.0
- **版本号**：V1.0
- **状态**：默认启用
- **可见性**：所有用户可见且可选择使用

---

## 二、需求目标

1. 实现**延迟初始化**：项目创建后不自动初始化里程碑，由用户在需要时主动触发
2. 提供**模板选择**：初始化时支持选择不同模板，也支持从零开始自定义
3. 支持**模板保存**：用户可将自定义里程碑保存为模板，支持公开/私有设置
4. 完善**模板管理**：系统管理员可在后台管理所有里程碑模板

## 三、需求详细说明

### 3.1 需求1：延迟初始化机制

#### 3.1.1 现状
- 项目创建时自动调用 `initializeProjectMilestones()` 初始化里程碑
- 所有项目使用统一的默认模板

#### 3.1.2 目标
- 项目创建成功后，**不自动初始化**里程碑
- 仅当用户点击进入项目的"里程碑"标签页时，才触发初始化流程

#### 3.1.3 业务规则
1. 新建项目时，`project_milestones` 表为空
2. 用户首次进入项目详情的"里程碑"页面时：
   - 检测到该项目无里程碑数据
   - 展示初始化引导界面（而非空白页面）
   - 引导用户选择初始化方式

#### 3.2.5 业务规则
1. **模板来源**：
   - **系统模板**（is_system = true）：由管理员创建，不可删除，所有用户可见

   - **用户私有模板**（is_system = false, is_public = false, created_by = 当前用户）：仅创建者可见
   - **用户公开模板**（is_system = false, is_public = true）：所有用户可见
   - 仅**启用的模板**（is_active = true）可被选择使用

2. **现有默认模板迁移**：
   - 升级时自动将现有默认模板转换为系统模板
   - 模板名称：标准项目里程碑模板 v1.0

   - 可见性：公开
   - 保留原有阶段和任务结构不变

3. **选择流程**：
   - 用户选择模板 → 点击"确认" → 系统根据模板创建项目里程碑
   - 或选择"从空白开始" → 进入空的里程碑页面，用户自行添加

4. **初始化后**：
   - 创建 `project_milestones` 记录
   - 创建 `milestone_tasks` 记录
   - 记录使用的模板ID（用于追溯）

4. **权限控制**：
   - 普通用户只能管理自己的模板
   - 管理员可以管理所有模板
   - 系统模板不可被普通用户修改/删除

#### 3.4.7 业务规则

1. **系统模板**（is_system = true）：
   - 由管理员创建和维护
   - **不可删除**（保护系统基础模板）
   - 仅管理员可编辑
   - 默认启用
   - 所有用户可见且可使用
   - **现有的"标准项目里程碑模板 v1.0"作为第一份系统模板保留**

2. **用户模板**（is_system = false）：
   - 创建者可编辑自己的模板
   - 管理员可编辑所有模板
   - 可删除（需确认）
   - 可设置为公开或私有

3. **启用/禁用**：
   - 禁用的模板不会在初始化时显示
   - 已使用禁用模板的项目不受影响
   - **系统模板至少保留一个启用状态**（防止无模板可用）
   - "标准项目里程碑模板 v1.0"默认始终启用

4. **权限控制**：
   | 角色 | 查看 | 创建 | 编辑 | 删除 | 启用/禁用 |
   |------|------|------|------|------|----------|
   | 普通用户 | 公开+自己 | 自己的 | 自己的 | 自己的 | - |
   | 管理员 | 全部 | 全部 | 全部 | 非系统 | 全部 |

5. **数据迁移说明**：
   - 升级时自动将现有 `template_versions` 和 `milestone_templates` 数据标记为系统模板
   - 现有模板的 `is_system` 字段设置为 true
   - 现有模板的 `is_public` 字段设置为 true
   - 确保历史数据平滑过渡

|------|--------|---------|
