#### 6. Bug修复 ✅

**问题**: API `/api/milestone-templates/available` 返回 500 错误
- **原因**: 查询使用了新字段（`is_system`, `is_public`, `tags` 等），但这些字段可能不存在于数据库中
- **解决**: 简化查询，只使用基本字段（`id`, `name`, `version_number`, `description`, `created_at`, `is_active`），在代码中添加默认值

**问题**: `userId` 类型错误（可能是 `undefined`）
- **解决**: 在调用 `initializeProjectMilestones` 前添加类型检查

**问题**: `useToast` hook 不存在
- **解决**: 使用 `alert()` 替代 `showToast()`

**问题**: 后端服务未重启导致代码未生效
- **解决**: 停止旧服务并重新启动

**问题**: 管理员模板管理页面无数据，API 返回 500 错误
- **原因**: 管理员 API 同样查询了不存在的新字段（`is_system`, `is_public`, `tags`, `profiles.full_name` 等）
- **解决**: 简化管理员 API 查询，移除对新字段和 profiles 表的依赖，在代码中添加默认值

**问题**: 版本管理创建失败，提示 `version_name` 字段违反非空约束
- **原因**: 数据库 `template_versions` 表中 `version_name` 字段是 `NOT NULL`，但代码插入时只提供了 `name` 字段
- **解决**: 在 `MilestoneTemplates.tsx` 中插入数据时同时提供 `version_name` 字段
