# 风险详情页字段编辑功能

## 功能概述

在风险详情弹窗中，支持对风险描述、影响分析、应对措施三个字段进行在线编辑。

### 主要改动

1. **新增图标导入**
   - 添加 `Edit2`, `Check`, `XCircle` 图标用于编辑操作

2. **新增状态管理**
   - `editingField`: 当前正在编辑的字段（'description' | 'impact' | 'mitigation_plan' | null）
   - `editValues`: 编辑中的字段值

3. **新增方法**
   - `startEditing(field)`: 开始编辑指定字段
   - `cancelEditing()`: 取消编辑
   - `saveEdit()`: 保存编辑内容到数据库

4. **UI 改造**
   - 风险描述、影响分析、应对措施三个区域添加"编辑"按钮
   - 点击编辑后显示 textarea 输入框
   - 提供"保存"和"取消"按钮
   - 编辑状态仅对具有编辑权限的用户显示

## 交互流程

1. 用户点击风险列表中的"详情"按钮打开弹窗
2. 在弹窗中点击任意字段的"编辑"按钮
3. 该字段变为可编辑的 textarea 输入框
4. 用户修改内容后点击"保存"或"取消"
5. 保存成功后更新本地状态和数据库

## 权限控制

- 仅当 `canEdit` 为 true 时显示编辑按钮
- 与新增风险、修改状态等操作保持一致的权限控制

## 技术实现

- 使用 Supabase 客户端直接更新 `risks` 表
- 更新成功后同步更新本地 state，无需重新加载列表
- 错误处理：保存失败时显示 alert 提示

## 测试建议

1. 验证三个字段均可正常编辑和保存
2. 验证取消编辑后内容保持不变
3. 验证保存后列表和弹窗内容同步更新
4. 验证无编辑权限用户看不到编辑按钮
