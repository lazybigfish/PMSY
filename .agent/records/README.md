# 项目记录文档目录

## 目录结构说明

本目录用于存放项目开发过程中的所有记录文档。文档按日期分类存放，便于查找和管理。

```
records/
├── 2026-02-12/          # 2026年2月12日的所有记录
├── 2026-02-13/          # 2026年2月13日的所有记录
└── README.md            # 本说明文件
```

## 文件命名规范

- **日期前缀**: `2026-02-13-` 或 `2026-02-12-`
- **内容描述**: 简短描述文档内容，如 `项目开发日报`、`功能实现记录`
- **文件后缀**: `.md`

## 示例

```
2026-02-13-项目开发日报.md
2026-02-13-功能实现记录.md
2026-02-12-需求分析.md
```

## 存放规则

1. **按日期分类**: 每个日期的文档放在对应的日期文件夹中
2. **每日汇总**: 建议每天创建一个日报文档汇总当天所有工作
3. **详细记录**: 具体功能实现、问题修复等可以单独创建文档
