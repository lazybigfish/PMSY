# PMSY 项目管理系统 - 设计系统规范

## 项目概述
- **产品类型**: SaaS 项目管理系统
- **风格定位**: 专业、现代、高效、清晰
- **设计模式**: 仪表盘式管理界面 (Dashboard Admin)
- **技术栈**: React + TypeScript + Tailwind CSS

## 6. 交互规范

### 过渡动画
| 元素 | 时长 | 缓动函数 | 属性 |
|------|------|----------|------|
| 按钮悬停 | 150ms | ease-in-out | background, transform |
| 卡片悬停 | 200ms | ease-out | box-shadow, transform |
| 模态框 | 300ms | cubic-bezier(0.4, 0, 0.2, 1) | opacity, transform |
| 下拉菜单 | 150ms | ease-out | opacity, transform |
| 页面切换 | 200ms | ease-in-out | opacity |

### 悬停效果
- **按钮**: 背景色变深, translateY(-1px)
- **卡片**: 阴影加深, translateY(-2px)
- **链接**: 颜色变为 Primary
- **表格行**: 背景变为 #F8FAFC

### 焦点状态
- **输入框**: 蓝色边框 + 蓝色光晕 (ring)
- **按钮**: 蓝色光晕 (ring-2)
- **链接**: 下划线

## 7. 响应式断点

| 断点 | 宽度 | 布局调整 |
|------|------|----------|
| Mobile | < 640px | 单列布局, 隐藏侧边栏, 汉堡菜单 |

| Desktop | 1024px - 1440px | 完整布局, 3列网格 |

## 10. 设计原则

### 清晰优先 (Clarity First)
- 信息层级分明，重要内容突出
- 使用足够的留白，避免视觉拥挤
- 文字对比度符合 WCAG AA 标准

### 一致性 (Consistency)
- 相同功能的组件外观一致
- 交互反馈模式统一
- 颜色、间距、字体遵循规范

### 效率导向 (Efficiency)
- 常用操作不超过 2 次点击
- 批量操作支持
- 键盘快捷键支持

### 反馈及时 (Feedback)
- 操作后立即给出视觉反馈
- 加载状态明确提示
- 错误信息清晰具体

---

## 11. 反模式 (Don'ts)

❌ **避免以下设计:**
- 使用 emoji 作为图标
- 按钮文字使用纯大写
- 阴影过重或颜色偏差
- 圆角不一致 (混合使用大圆角和小圆角)
- 文字对比度不足
- 可点击元素没有 cursor-pointer
- 表单标签使用占位符替代
- 动画时间过长 (>500ms)
- 布局跳动 (Layout Shift)
